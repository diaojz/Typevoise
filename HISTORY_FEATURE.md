# 历史记录功能说明

## 功能概述

历史记录功能允许用户查看、搜索和管理所有的语音转文字记录。每次使用快捷键进行语音输入后，系统会自动保存原始识别文本和 AI 润色后的文本。

## 主要特性

### 1. 自动保存
- 每次语音转文字完成后自动保存
- 保存内容包括：
  - 原始识别文本
  - AI 润色后的文本
  - 时间戳
  - 唯一标识符

### 2. 历史记录查看
- 双栏布局：左侧列表 + 右侧详情
- 列表显示：
  - 相对时间（刚刚、5分钟前、昨天等）
  - 绝对时间（精确到秒）
  - 润色后的文本预览（最多2行）
  - 原始文本预览（如果与润色文本不同）
- 详情显示：
  - 完整的润色文本
  - 完整的原始文本
  - 精确时间戳

### 3. 搜索功能
- 实时搜索
- 同时搜索原始文本和润色文本
- 不区分大小写

### 4. 复制功能
- 右键菜单快速复制
- 详情页面复制按钮
- 可分别复制原文或润色文本

### 5. 删除功能
- 删除单条记录（右键菜单）
- 清空所有记录（工具栏按钮）
- 删除前有确认提示

## 数据存储

### 存储位置
```
~/Library/Application Support/Typevoise/history.json
```

### 数据格式
```json
[
  {
    "id": "UUID",
    "timestamp": "2026-04-04T10:30:00Z",
    "originalText": "原始识别文本",
    "polishedText": "润色后的文本"
  }
]
```

### 数据持久化
- 使用 JSON 格式存储
- 每次添加或删除记录时自动保存
- 应用启动时自动加载

## 使用方法

### 打开历史记录
1. 点击主界面的"历史记录"按钮
2. 或使用菜单栏快捷方式（如果配置）

### 查看记录
1. 在左侧列表中选择一条记录
2. 右侧会显示完整的详情信息

### 搜索记录
1. 在顶部搜索框输入关键词
2. 列表会实时过滤显示匹配的记录

### 复制文本
**方式 1：右键菜单**
1. 在列表中右键点击记录
2. 选择"复制原文"或"复制润色文本"

**方式 2：详情页面**
1. 在右侧详情页面点击对应的"复制"按钮

### 删除记录
**删除单条：**
1. 在列表中右键点击记录
2. 选择"删除"
3. 确认删除

**清空所有：**
1. 点击工具栏的"清空"按钮
2. 确认清空

## 技术实现

### 架构设计
```
TranscriptionRecord (Model)
    ↓
HistoryManager (Manager)
    ↓
HistoryView (View)
```

### 核心组件

#### TranscriptionRecord
- 数据模型
- 遵循 Identifiable, Codable, Hashable 协议
- 提供格式化时间显示

#### HistoryManager
- 单例模式
- 遵循 ObservableObject 协议
- 负责数据的增删改查和持久化

#### HistoryView
- SwiftUI 视图
- NavigationView 双栏布局
- 支持搜索、选择、上下文菜单

### 数据流
```
VoiceController
    ↓ (语音识别完成)
ClaudeService
    ↓ (AI 润色完成)
HistoryManager.addRecord()
    ↓ (保存到文件)
history.json
    ↓ (通知视图更新)
HistoryView (自动刷新)
```

## 隐私和安全

### 本地存储
- 所有数据仅存储在本地
- 不会上传到任何服务器
- 用户完全控制自己的数据

### 数据访问
- 仅应用本身可以访问
- 存储在应用的沙盒目录中
- 受 macOS 权限系统保护

### 数据清理
- 用户可以随时删除单条记录
- 用户可以随时清空所有记录
- 卸载应用时数据会被自动删除

## 常见问题

### Q: 历史记录会占用多少空间？
A: 每条记录大约 100-500 字节，1000 条记录约 100-500 KB，占用空间很小。

### Q: 历史记录有数量限制吗？
A: 目前没有硬性限制，但建议定期清理不需要的记录以保持应用性能。

### Q: 如何备份历史记录？
A: 可以手动复制 `~/Library/Application Support/Typevoise/history.json` 文件进行备份。

### Q: 卸载应用后历史记录会保留吗？
A: 不会，卸载应用时历史记录会被一起删除。如需保留，请先备份 history.json 文件。

### Q: 可以在多台设备间同步历史记录吗？
A: 目前不支持，历史记录仅存储在本地。未来可能会添加 iCloud 同步功能。
